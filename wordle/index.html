My Wordle solvers page

This is my page for some Wordle solvers in different programming languages.

Description of the approach used

All programs use the same approach for solving a Wordle problem:
  • Get the words that satisfies the three constraints (hints) in Wordle:
    • Characters in correct positions
    • Characters that are in the word but not in correct position
    • Characters not in the word
  • Then sort these words according to the scoring system which is based on the frequencies of each positions of the words in some wordlist.
    The frequency table looks like this:
    • position 1: "xzyjkquinovhewlrmdgfaptbcs"
    • position 2: "jzqfkgxvbsdymcwptnhulieroa"
    • position 3: "qjhzkxfwyvcbpmgdstlnrueoia"
    • position 4: "jqxyzbwhfvpkmdguotcrilasne"
    • position 5: "jqvuzxbiwfcsgmpoakdnhlrtye"
    The lists are sorted in increasing order of frequency. The scoring of a word is then the sum of the words characters positions in each of these frequency lists. Also, if a word has 5 unique characters then it get 100 more points.

The programs

Here are my Wordle solvers (so far), grouped according to the programming language used.
  • Picat:
    • wordle.pi: Solver, tester and quite a few experiments. This was the first Wordle solver I wrote.
    • wordle2.pi: This has just the solver part from wordle2.pi. It's the fastest of the Picat programs.
    • wordle3.pi: Experimental: Using cl_facts to load the words and then it use simple Prolog style for getting the words.
    • wordle4.pi: Using plain foreach loops.
    • wordle_cp.pi: Just for fun: Constraint modelling. Using the cp module.
    • wordle_dcg.pi: Even more fun: Using DCG, including a DCG for generating the wordlist words.
  • Python The program wordle.py has these four methods:
    • wordle1: Using z3's String constraint.
    • wordle1b: Using z3's String constraint, faster approach.
    • wordle2: Plain Python approach, no regexes.
    • wordle3: Plain Python program, using regexes. This is the fastest of the Python methods.
  • Julia
    • wordle.jl: Using Regexes
  • K (ngn)
    • wordle.k: The shortest program of them all.
  • Mathematica / Wolfram Language
    • wordle_solver.nb: Mathematica notebook.
  • Perl)
    • wordle.pl: Using regexes.
  • SWI-Prolog)
    • wordle.pl: This is a port of my Picat program wordle2.pi to SWI-Prolog
  • Unicon)
    • wordle.icn: Using regexes.
